# 常見問題（FAQ）

這份 FAQ 整理測試 **Android 11+ DAP SD 卡儲存行為** 時最常遇到的問題。

[返回主 README](../README.md) · [繁中正文](README.zh-TW.md) · [简体中文 FAQ](faq.zh-CN.md) · [English FAQ](faq.md) · [相容性矩陣](compatibility.md)

---

## 1. SD 卡明明有被辨識，為什麼 App 還是在吃內建儲存？

因為 SD 卡辨識和 App 資料寫入不是同一層。

在 Android 11+ 上，系統可以看得到 SD 卡，設定畫面也可以允許搬移 App，但 App 自己的離線歌曲或快取仍可能寫在內建共享儲存層。這份 repo 會把這些層分開看。

---

## 2. App 已經搬移成功，為什麼資料沒有跟著搬？

App 本體位置和 App 離線資料位置不一定相同。安裝包可以搬到外接卷，但下載資料或快取仍可能留在：

```text
/storage/emulated/0
```

這就是 Sony Walkman + QQ 音樂案例中觀察到的主要現象。

---

## 3. 把 App 搬到 SD 卡，離線音樂也會一起搬嗎？

不一定。

已驗證的 Walkman + QQ 音樂測試結果是：

```text
App 本體搬移 -> 有
離線歌曲跟著搬移 -> 沒有
```

所以 App 搬移成功不能直接當成離線資料搬移成功。

---

## 4. 為什麼 Windows / MTP 還是只看到大約 103GB，不是完整 1TB？

Windows / MTP 通常看到的是共享儲存層，不是 Android 底下每一個外接或擴充卷。

在這次測試中，Walkman 設定頁可能顯示較大的總容量，但 Windows 仍主要看到內建共享儲存那一側。這不代表 SD 卡沒生效，只代表它不是用一般外接碟的方式暴露給 MTP。

---

## 5. `/storage/emulated/0` 是什麼？

它是使用者最常在檔案管理器看到的內部共用儲存層。很多 App 也會把它顯示成下載位置。

它很重要，但不代表整台裝置所有可用儲存都在這裡。

---

## 6. `/mnt/expand/<UUID>` 是什麼？

它是 Android 底層看到的外接卷或擴充卷路徑。在某些情況下，Package Manager 可能把 App 本體放到這裡。

但即使 App 本體在這裡，也不代表離線歌曲或快取資料一定會一起進來。

---

## 7. `disk:179,192` 是什麼意思？

它是 Android `StorageManager` 回報的磁碟裝置代號，常見於：

```bash
adb shell sm list-disks
```

它不是掛載路徑，也不是內建儲存本體，只是系統用來識別磁碟的代號。

---

## 8. 一定要 root 嗎？

不用。這份 repo 驗證過的主流程是 non-root。

一般路線是：開 USB 偵錯，用 ADB 觀察，安裝並執行一次 App，用系統 UI 手動搬移 App，最後再用 ADB 驗證 App 本體和資料行為。

---

## 9. 下面這些指令一定要跑嗎？

```bash
adb shell sm set-force-adoptable true
adb shell sm partition disk:179,192 private
adb reboot
adb shell sm list-volumes all
```

不用。它們屬於研究歷程，不是一般讀者的主流程。

這些指令有助於觀察底層儲存行為，但可能牽涉儲存重建或格式化風險。這份 repo 的穩定主線仍是 UI 手動搬移 App，再用 ADB 驗證。

---

## 10. 這篇的主流程到底是 UI 還是指令？

搬移動作是 UI 主導，驗證動作是 ADB 主導。

實際操作是先安裝 App、執行一次、關閉，再進入：

```text
設定 → 儲存空間 → 應用程式 → 儲存位置
```

手動搬移後，再用 ADB 看實際變化。

---

## 11. 為什麼要先安裝 App、執行一次，再關閉？

這樣 App 才有一個比較完整的初始化狀態。沒有先執行一次，package 狀態和資料行為可能不完整，後面的 ADB 比對也比較沒有意義。

---

## 12. 為什麼 QQ 音樂可以搬，離線歌曲卻還是在內建？

因為系統能搬的是 App 本體，App 自己仍可能把離線歌曲寫進共享儲存層。

這不是矛盾，而是 App 本體位置和 App 資料行為分開。

---

## 13. 「修改系統設定」權限有關嗎？

QQ 音樂可能有這樣的入口：

```text
應用程式資訊 → 進階 → 修改系統設定
```

就這次儲存行為測試來看，目前沒有足夠證據顯示這個權限會直接決定離線歌曲能不能搬到外接儲存。它可以記錄為觀察項，但不適合直接當主因。

---

## 14. 這個問題靠 AI 能解決嗎？

AI 可以幫忙拆層、設計驗證流程、解讀 ADB 結果，也可以把測試紀錄整理成比較清楚的文件。

但 AI 不能改寫 Android 的儲存規則，也不能保證每個 App 都會把離線資料寫到使用者想要的位置。這件事最後仍要靠裝置端驗證。

---

## 15. 現在最實際可行的解法是什麼？

以 Sony Walkman + QQ 音樂案例來說，目前比較實際的角色分工是：

```text
下載 -> 內建儲存
存放 -> SD 卡
播放 -> 從 SD 卡讀
```

也就是把內建儲存當系統 / App 盤，把 1TB SD 卡當音樂資料盤。必要時仍需要手動搬移音樂資料夾。

---

## 16. 這個 repo 是不是對所有 DAP 和所有串流 App 都成立？

不是。它目前是一個 Sony Walkman 已驗證案例、一套可重現方法，以及後續相容性回報入口。

其他裝置與 App 請看 [相容性矩陣](compatibility.md)。目前沒驗證過的項目，都應視為待驗證問題，不是既定事實。
